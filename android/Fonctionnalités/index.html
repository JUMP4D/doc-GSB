    Fonctionnalités de l'API de GSB VISITES

    L'application GSB VISITES utilise une API REST. Cette API permet de gérer les différentes fonctionnalités de l'application liées à l'authentification, la création de rapport de visite et autres fonctionnalité de l'application.

    1. Authentification des utilisateurs

    L'application permet aux utilisateurs (visiteurs médicaux) de s'authentifier via l'API.

    • Route de l'API : POST /api/visiteur/login

    Formulaire d'authentification :

    • email : [Champ de saisie] L'adresse email de l'utilisateur.
    • password : [Champ de saisie] Le mot de passe de l'utilisateur.

    Réponse :

    • id : Identifiant de l'utilisateur.
    • token : Un jeton d'authentification JWT pour sécuriser les requêtes suivantes.

    2. Création d'un rapport

    L'application permet à l'utilisateur de créer un rapport de visite.

    Formulaire Rapport de visite :

    • date : [Champ de saisie] Date de la visite.
    • commentaire : [Champ de saisie] Commentaire détaillant la visite.
    • praticien : [Liste déroulante] Sélection du praticien (praticiens disponibles).
    • motif : [Liste déroulante] Sélection du motif de la visite.

    Route de l'API : POST /api/visite

    3. Consultation des visites

    L'utilisateur peut consulter les visites qu'il a effectuées.

    • Route de l'API : GET /api/visites

    Réponse :

    Liste des visites avec les informations suivantes :

    • date : Date de la visite.
    • commentaire : Commentaire ajouté lors de la visite.
    • motif : Motif de la visite.

    4. Ajout d'un praticien

    L'application permet à l'utilisateur d'ajouter un praticien à son portefeuille.

    Formulaire Praticien :

    • nom : [Champ de saisie] Nom du praticien.
    • prenom : [Champ de saisie] Prénom du praticien.
    • tel : [Champ de saisie] Numéro de téléphone du praticien.
    • email : [Champ de saisie] Adresse email du praticien.
    • rue : [Champ de saisie] Adresse de la rue du praticien.
    • code_postal : [Champ de saisie] Code postal du praticien.
    • ville : [Champ de saisie] Ville du praticien.

    Route de l'API : POST /api/praticien

    5. Consultation des praticiens

    L'utilisateur peut consulter la liste des praticiens de son portefeuille.

    • Route de l'API : GET /api/praticiens/{id}

    Réponse :

    Liste des praticiens avec les informations suivantes : - nom : Nom du praticien. - prenom : Prénom du praticien. - tel : Numéro de téléphone. - email : Adresse email.
