    Sécurité de GSB VISITES

    Bibliothèques JavaScript utilisées

    1. Express Validator
      Utilisée pour valider les données envoyées par l'utilisateur (par exemple, vérifier si un email est valide ou si un mot de passe respecte les critères de sécurité).

    2. Mongoose Encryption
      Permet de crypter les données sensibles (comme les mots de passe) avant de les stocker dans la base de données.

    3. bcrypt
      Utilisée pour le hachage des mots de passe, assurant qu'ils ne sont jamais stockés en clair dans la base de données.

    4. express-rate-limit

      Utilisé pour limiter le nombre de requêtes effectuées dans un délai donné, protégeant ainsi l'API contre les attaques par brute force ou les attaques par déni de service (DoS).

    Authentification et JWT

    L'authentification dans l'application est assurée via un JWT (JSON Web Token). Après l'authentification réussie de l'utilisateur (avec email et mot de passe), un token JWT est généré pour sécuriser les requêtes suivantes.

    • Méthode d'authentification : POST /api/visiteur/login
    • Middleware d'authentification : Un middleware vérifie la présence et la validité du token JWT dans les en-têtes des requêtes. Cela garantit que seuls les utilisateurs authentifiés peuvent accéder aux ressources protégées de l'application.

    Code du middleware de limite de requete

    import rateLimit from "express-rate-limit";
import { Request, Response, NextFunction } from "express";
import bcrypt from "bcrypt";
import Visiteur from "../models/visiteurModel";

// Configuration du middleware de limitation de débit
const limiter = rateLimit({
    windowMs: 5 * 60 * 1000, // Fenêtre de temps de 5 minutes
    max: 3, // Limite chaque IP à 3 requêtes par fenêtre de 5 minutes
    message: "Trop de tentatives de connexion échouées depuis cette adresse IP, veuillez réessayer plus tard.", // Message renvoyé lorsque la limite est atteinte
});

export const loginRateLimiter = async (req: Request, res: Response, next: NextFunction) => {
    const { email, password } = req.body;

    try {
        // Recherche du visiteur par email
        const visiteur = await Visiteur.findOne({ email });
        if (!visiteur) {
            // Si le visiteur n'existe pas, appliquer la limitation de débit
            return limiter(req, res, next);
        }

        // Vérification du mot de passe
        const isPasswordValid = await bcrypt.compare(password, visiteur.password);
        if (!isPasswordValid) {
            // Si le mot de passe est incorrect, appliquer la limitation de débit
            return limiter(req, res, next);
        }

        // Si les informations de connexion sont correctes, passer au middleware suivant
        next();
    } catch (err) {
        // Gestion des erreurs
        res.status(500).json({ message: "Erreur lors de la vérification des informations de connexion.", error: err });
    }
};

    Code du middleware d'authentification

    import { Request, Response, NextFunction } from 'express';
import jwt from 'jsonwebtoken';

// Étendre l'interface Request pour inclure la propriété auth
declare module 'express-serve-static-core' {
  interface Request {
    auth?: string;
  }
}

// Interface pour le token décodé
interface DecodedToken {
    VisiteurId: string;
}

// le middleware vérifie si l'utilisateur est authentifié
export const authMiddleware = (req: Request, res: Response, next: NextFunction): void => {
    try {
      const authHeader = req.headers.authorization;
      if (!authHeader) {
        throw new Error('Authorization header is missing.');
      }

      const token = authHeader.split(' ')[1];
      const decodedToken = jwt.verify(token, process.env.JWT_SECRET as string) as DecodedToken;

      req.auth = decodedToken.VisiteurId;
      next();
    } catch (error) {
      res.status(401).json({ error: 'Unauthorized request' });
    }
  };
